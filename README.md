# UE5 CSV Profile Viewer

A desktop application for visualizing and comparing Unreal Engine 5 CSV profiling dumps. Load frame-by-frame performance data exported from UE5, explore timing, memory, rendering, and physics metrics on an interactive graph, and optionally overlay a second capture for side-by-side comparison.

## Features

- **Interactive graph** — zoom (scroll wheel), pan (middle-click drag), and region-select zoom (left-click drag) across thousands of frames.
- **Channel layer panel** — toggle individual metrics on/off, grouped by category (Timing, Memory, CPU, GPU, Rendering, Physics, etc.). Includes quick-toggle buttons for "All On", "All Off", and "Timing only".
- **Comparison mode** — load a second CSV to overlay it on the primary capture with dashed lines. Use the shift slider (with single-frame arrow buttons) to align the two captures in time.
- **Frame detail inspector** — click any frame to view a full breakdown across Timing, Threads, Memory, Rendering, and Physics tabs.
- **Smoothing** — apply a moving-average filter to reduce noise and spot trends.
- **Vertical scale** — dedicated slider on the left edge of the graph to zoom the Y-axis.
- **Scrollbars** — horizontal and vertical scrollbars at the graph edges for mouse-free navigation.
- **Crosshair & tooltip** — hover over the graph to see per-channel values at the current frame, including comparison deltas when a second CSV is loaded.
- **FPS target lines** — reference lines at 16.67 ms (60 FPS) and 33.33 ms (30 FPS).

## Requirements

- Python 3.10+
- PySide6
- pyqtgraph
- NumPy

Install dependencies:

```bash
pip install PySide6 pyqtgraph numpy
```

## Usage

### Running from source

```bash
python main.py
```

### Building the native C++ acceleration module (optional)

The application includes an optional C++ extension (via pybind11) that significantly speeds up CSV parsing, data smoothing, and graph rendering for large files. It requires a C++ compiler (MSVC on Windows, gcc/clang on Linux/macOS):

```bash
pip install pybind11
python setup.py build_ext --inplace
```

The application works without the native module (pure-Python fallback), but building it is recommended when working with large CSV files.

### Building a standalone executable

```bash
python build.py
```

This produces `dist/UE5_CSV_Viewer.exe` via PyInstaller (single-file, windowed).

### Loading data

1. Click **Load CSV** and select a UE5 CSV profiling dump.
2. The graph populates with all available channels. By default only timing channels are enabled.
3. Use the **layer panel** on the left to toggle channels, filter by name, or use the quick-select buttons.

### Comparing two captures

1. Click **Load Comparison CSV** to overlay a second profile.
2. The comparison appears as dashed lines in the same colors.
3. Use the **Shift** slider (or the arrow buttons) below the graph to offset the comparison in time. Each arrow click shifts by exactly one frame.
4. Hover over any frame to see per-channel deltas between the two captures.
5. Click **Clear Comparison** to remove the overlay.

### Navigating the graph

| Action | Control |
|---|---|
| Zoom in/out | Scroll wheel |
| Pan | Middle-click drag |
| Region zoom | Left-click drag a rectangle |
| Select frame | Left-click |
| Reset view | "Reset Zoom" button |
| Scroll horizontally | Horizontal scrollbar at bottom |
| Scroll vertically | Vertical scrollbar at right |
| Scale Y-axis | Vertical slider at left edge |

### Inspecting a frame

Click on any frame in the graph. The **Frame Details** panel on the right shows categorized data across tabs:

- **Timing** — FrameTime, GameThreadTime, RenderThreadTime, GPUTime, etc.
- **Threads** — per-thread exclusive timings.
- **Memory** — free, physical, virtual, GPU memory, texture streaming.
- **Rendering** — draw calls, primitives, RDG counts, lights, shadows, PSO misses, shaders.
- **Physics/Ticks** — Chaos physics and tick counts.
- **All Data** — every channel value for the selected frame.

## CSV format

The application expects UE5's built-in CSV profiling format:

- First row: comma-separated headers starting with `EVENTS`.
- Data rows: first column is event text (usually empty), remaining columns are numeric values.
- Optional trailing metadata rows marked with `[HasHeaderRowAtEnd]`.

These files are typically generated by the `stat startfile` / `stat stopfile` console commands in UE5, or via `FPSChartStart` / `FPSChartStop`.
